# plan_0016 - 5시퀀스 평가 시나리오 생성/주입 및 BE-router 경유 테스트

## 상태
- 진행 중 (2026-02-08)

## 목표
- 최소 5개 아이템, 5개 NPC, 5개 적을 포함한 5시퀀스 평가 시나리오를 생성한다.
- 시나리오 주입은 예외로 직접 수행하고, 그 외 테스트 흐름은 BE-router를 통해 실행한다.
- BE-router의 세션 시작으로 주입된 시나리오를 복제/시작하여 턴 테스트를 수행한다.

## 작업 단위
1. 시나리오/프로파일 자산 추가
- `tester/src/tester/scenario_payloads`: 5시퀀스 주입 payload 추가
- `tester/src/tester/scenario_profiles`: 평가 프로파일 추가
- `tester/src/tester/scenarios.py`: 신규 프로파일 로더 등록

2. 시나리오 주입
- `tester/src/tester/seed_profile.py`를 사용해 payload를 scenario-service debug inject-save로 주입
- lock 파일 생성으로 state scenario id 고정

3. BE-router 경유 테스트
- `PYTHONPATH=tester/src uv run python -m tester.runner <sid> <turns> <profile>`
- 세션 시작(`/state/session/start`), 턴 처리(`/gm/turn`) 등은 BE-router 경유

4. 결과 분석/문서화
- 시퀀스 전이, 등장/퇴장, 상태 업데이트(아이템/엔티티) 근거를 로그에서 정리

## 완료 기준 (DoD)
- 신규 평가 시나리오가 주입되고 lock id가 생성된다.
- 테스트 실행이 BE-router 경유로 수행된다.
- 결과 분석이 `docs/dev/detail`에 기록된다.
