# plan_0009 - Session Transition 및 State 반영 정렬

## 상태
- 완료 (2026-02-07)

## 목표
- `scenario-service`의 `/api/v1/manage/sessions/transition`를 실제 동작으로 구현한다.
- 액트/시퀀스 전이 요청 시 state-manager 세션 상태가 즉시 갱신되도록 정렬한다.

## 범위
- 수정 대상: `scenario-service`만.
- 금지 범위: `rule-engine` 코드 변경 금지, `state-manager` 직접 수정 금지.

## 작업 단위
1. Transition 계약 정리
- `next_act_id`는 선택, `next_seq_id`는 필수로 처리
- ID 입력(`act_2`, `seq_5`)을 canonical(`act-2`, `seq-5`)로 정규화

2. 전이 라우팅 구현
- `next_act_id` 존재: state-manager `PUT /state/session/{session_id}/act`
- `next_act_id` 없음: state-manager `PUT /state/session/{session_id}/sequence`

3. 오류 경계 정렬
- 잘못된 ID 형식: 400
- state-manager upstream 실패: 502
- 기타 예외: 500

4. 테스트 증빙
- 액트+시퀀스 전이 성공 케이스
- 시퀀스 단독 전이 성공 케이스
- 잘못된 시퀀스 ID 실패 케이스

## 완료 기준 (DoD)
- transition API가 더 이상 더미 응답이 아니고 state-manager 갱신 요청을 실제 전송해야 함.
- 전이 입력 ID 형식이 canonical 규칙으로 정렬되어야 함.
- 상기 3개 테스트 시나리오가 자동화 테스트로 통과해야 함.
