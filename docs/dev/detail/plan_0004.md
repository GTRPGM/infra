# plan_0004 - BE-router 및 연동 경계 정렬

## 목표
- BE-router가 GM 및 기타 서비스 계약을 왜곡 없이 중계하도록 정렬한다.

## 작업 단위
1. GM 프록시 계약 점검
- 대상: `services/BE-router/src/gm/gm_routers.py`
- 조치:
  - 요청 body/헤더(auth)/응답 전달 형식 일치 확인
  - 불필요한 변환 제거(원형 전달)

2. State/Info/Minigame 경계 점검
- 대상: `services/BE-router/src/state/*`, `services/BE-router/src/info/*`, `services/BE-router/src/minigame/*`
- 조치:
  - 내부 서비스 계약과 다른 래핑 여부 점검
  - 에러코드 전달 일관성 확보

3. DTO 정렬
- 대상: `services/BE-router/src/*/dtos`
- 조치:
  - 실제 하위 서비스 계약과 불일치하는 필드 정리

## 완료 기준 (DoD)
- BE-router 경유와 직접 호출 결과가 의미적으로 동일해야 함(상태코드/핵심 payload 기준).

## 실행 결과 (2026-02-07)
- GM 프록시 계약 점검/정렬 완료:
  - `gm_routers.py`의 디버그 `print()` 제거.
  - `/gm/turn` 중계 시 요청 body(`session_id`, `content`) 원형 전달 확인.
- 공통 프록시 에러 전달 정렬:
  - `proxy_request`가 upstream 오류 응답에서 `detail/message`를 보존해 `HTTPException`으로 전달.
  - 성공 응답이 비JSON일 때도 안전 fallback(`{"raw": ...}`) 처리.
