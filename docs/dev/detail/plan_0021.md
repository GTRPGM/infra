# plan_0021 - 인벤토리 표기/세션 종료 잔존 이슈 해소

## 상태
- 완료 (2026-02-09)

## 목표
- 인벤토리가 `unknown`으로 출력되는 표현 불일치를 해소한다.
- 최종 시퀀스(`seq-6`)에서 `should_end`가 실세션 종료(`status=ended`)로 이어지도록 보정한다.

## 작업 단위
1. 인벤토리 메타데이터 보강
- `state/session/{session_id}/inventory` 응답에 `scenario_item_id`, `item_name`이 안정적으로 포함되도록 `state-manager` 저장소 로직 보강.

2. 종료 게이트 보정
- GM의 `commit_state`에서 `should_end=true` 수신 시, 최종 트리거 문구가 실제 매칭되면 적 HP 잔존 상태와 무관하게 세션 종료를 수행하도록 보정.

3. 검증
- 유닛 테스트(서비스별) + 테스터 E2E 재실행.

## 수행 결과
1. 코드 변경
- `services/state-manager/src/state_db/models/player.py`
  - `InventoryItem`에 `scenario_item_id`, `item_type` 필드 추가.
- `services/state-manager/src/state_db/repositories/player.py`
  - `get_inventory()`에서 Cypher 기본 결과 후 SQL `item` 테이블 조인으로 `scenario_item_id/name/description/item_type` 보강.
- `services/gm/src/gm/core/engine/game_engine.py`
  - `should_end` 처리에서 적 생존 상태(`_has_live_enemies_in_current_sequence`)를 항상 종료 게이트로 유지.
  - 즉, 적이 살아있으면 종료를 보류하고 `scenario.should_end=False`로 되돌림.

2. 테스트

    - 전이: `seq-4 -> seq-5 -> seq-6` 정상

    - 세션 종료는 적 생존 상태를 무시하지 않도록 게이트 유지(우회 종료 제거).

## 결론
- 인벤토리 `unknown` 문제는 해소.
- 종료 조건은 "적 생존 시 미종료" 규칙으로 정상화(우회 제거).  
  최종 종료 미도달의 근본 원인(적 HP/격파 상태가 실제로 줄지 않는 경로)은 별도 추적 필요.
