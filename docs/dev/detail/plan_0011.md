# plan_0011 - 시퀀스 전이 누락 탐지 강화

## 상태
- 완료 (2026-02-07)

## 목표
- 서술상 `exit_trigger` 충족 정황이 있는데도 시퀀스가 유지되는 문제를 감지/재현하고 원인을 제거한다.
- 테스터는 오탐 없이 전이 누락을 검출해야 한다.

## 범위
- 수정 대상: `tester/src/tester/runner.py`, `tests/test_plan_0010_context_assertions.py`, `services/scenario-service/src/scenario/core/engine/scenario_engine.py`, `services/scenario-service/tests/test_plan_0011_transition_fallback.py`, `docs/dev/*`
- 비대상: `rule-engine`, `state-manager` 직접 수정

## 작업 단위
1. 원인 분석
- `scenario-service` validator 프롬프트 기대 입력(`act_id`, `current_sequence_id`, `available_sequences`)과 실제 전달 데이터 불일치 확인
- `next_seq_id` 미반환 시 GM 커밋 단계에서 전이 스킵되는 경로 확인

2. 트리거 감지 로직 보강 (Tester)
- 기존: action에 trigger 문구가 부분 포함될 때만 감지
- 변경: action+narrative 결합 텍스트 기준으로
  - 정규화 문구 포함 검사
  - 키워드 겹침 비율 기반 검사(휴리스틱)

3. 전이 보정 로직 추가 (Scenario Service)
- validator 입력에 `act_id`, `current_sequence_id`, `available_sequences`를 추가
- validator가 `next_seq_id`를 못 주더라도, `exit_triggers`와 `user_input` 휴리스틱 매칭 시 다음 시퀀스를 fallback으로 지정

4. 오탐 방지 정밀화 (Tester)
- trigger 매칭 입력을 `player_action` 중심으로 제한
- narrative는 “완료형 진행 힌트”만 전이 정황으로 판정
- 의도/계획 표현(예: `~위해`, `~하려`)은 진행 힌트에서 제외

5. 회귀/재현 검증
- 단위 테스트: trigger 전이 누락 실패 케이스 유지
- 통합 스모크: 수정 전 실패/수정 후 재실행 비교
