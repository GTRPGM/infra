# plan_0019 - 인벤토리 경로 정렬 및 E2E 재검증

## 상태
- 완료 (2026-02-08)

## 목표
- 아래 3개 수정사항을 적용한다.
  1) BE-router에 `/state/player/item/earn`, `/state/player/item/use` 프록시 추가
  2) 테스터 starter loadout 경로를 `inventory/update(rule_id)`에서 `player/item/earn(item_uuid)`로 전환
  3) `inventory/update`는 no-op를 제거하고 실제 반영 또는 실패 반환으로 정렬
- 유닛 테스트를 실행해 수정사항을 검증한다.
- 고도화 시나리오 테스터를 재실행해 전체 진행/상태 갱신을 확인한다.

## 작업 단위
1. API/모델 정렬
- state-manager `inventory/update` no-op 처리 개선
- BE-router state DTO/라우터에 item earn/use 프록시 추가

2. 테스터 경로 전환
- 세션 시작 후 starter 지급 시 item UUID 조회 후 `/state/player/item/earn` 사용

3. 단위 검증
- state-manager/BE-router/테스터 관련 유닛 테스트 실행

4. 통합 검증
- `six_sequence_three_act_eval` 시나리오 주입 + 러너 실행
- 시퀀스 전이/인벤토리/엔티티 상태 관측

## 완료 기준 (DoD)
- 3개 수정사항 코드 반영 완료
- 유닛 테스트 통과 로그 확보
