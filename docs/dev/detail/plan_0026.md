# plan_0026 - 종료 게이트 정합성 및 터미널 반복 보강

## 상태
- 완료 (2026-02-09)

## 목표
- `should_end=true`여도 현재 시퀀스 적이 생존하면 세션 종료를 차단한다.
- 마지막 시퀀스에서 적이 모두 처치되었는데도 종료 신호가 누락되는 반복 케이스를 상태 기반으로 종료 보정한다.
- 테스터 결과에 현재 시퀀스 생존 적 수를 포함해 회귀 판정 신뢰도를 높인다.

## 작업 단위
1. GM `commit_state` 종료 경로에서 적 생존 검증을 강제한다.
2. GM에 터미널 시퀀스 + 무생존 적 조건의 상태 기반 종료 보조 로직을 추가한다.
3. 러너 결과 JSON에 `alive_enemies_in_current_sequence`를 추가한다.
4. 관련 단위 테스트 및 E2E 재검증을 수행한다.

## 수행 결과
- 반영 완료:
  1. `gm` 종료 게이트 보강
     - `should_end=true` 시 현재 시퀀스 적 생존(`HP>0` 또는 `is_defeated=false`)이면
       `end_session` 차단 유지.
  2. `gm` 터미널 반복 종료 보조
     - `should_end=false`여도 다음 조건 충족 시 자동 종료:
       - 현재 시퀀스가 현재 Act의 마지막 시퀀스
       - 현재 시퀀스 적이 1명 이상 존재
       - 현재 시퀀스 생존 적 수가 0
  3. 테스터 러너 지표 보강
     - 턴별/최종 리포트에 `alive_enemies_in_current_sequence` 추가.
