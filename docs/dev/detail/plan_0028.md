# plan_0028 - 서사-상태 정합 가드레일

## 상태
- 완료 (2026-02-09)

## 목표
- GM 서사가 상태보다 앞서 “종료/완료/전원 처치”를 선언하지 않도록 가드레일을 추가한다.
- 서사 생성은 상태 결과를 설명하는 역할로 제한하고, 판단 권위를 가지지 않도록 정렬한다.

## 작업 단위
1. GM `generate_narrative`에서 post-commit 상태(적 생존 여부)를 재조회한다.
2. 생존 적이 있으면 종료/완료 선언 문구를 금지하고 재시도한다.
3. 충돌 상황에서 필수 문구 강제(`required_slot`)가 서사-상태 모순을 만들지 않도록 완화한다.
4. 관련 단위 테스트를 추가하고 E2E 로그로 정합성 재확인한다.

## 수행 결과
1. `services/gm/src/gm/core/engine/game_engine.py`
   - post-commit 스냅샷 기준 생존 적 존재 시 종료/완료 서사 금지 패턴을 확장했다.
   - 프롬프트 컨텍스트(`world_snapshot.enemies`)에 `hp`, `is_defeated`, `assigned_sequence_id`를 포함해 LLM이 상태를 직접 참조하게 했다.
   - 내러티브 생성 컨텍스트에서 `exit_triggers`를 제거하고, `scenario_suggestion.narrative_slot` 강제를 제거해 GM이 전이 조건 문구를 직접 참조하지 않도록 분리했다.
   - 종료/완료 서사가 반복 재시도 후에도 남는 경우, 부분 치환 대신 상태기반 안전 문장으로 강제 대체하도록 변경했다.
   - 종료 문구 `"모험은 끝이 났다."`는 LLM 유도가 아니라 `session.status=ended` + 생존 적 없음일 때만 후처리로 부여하도록 변경했다.

2. 테스트
   - 단위: `services/gm/tests/test_narrative_guardrails.py` 신규 케이스 포함 `3 passed`.
   - 회귀: `tests/test_plan_0003_contract_alignment.py tests/test_pipeline_precision.py tests/test_game_summary.py` `18 passed` (기존 warning 1건 유지).

3. E2E 검증 (BE-router 경유)

     - HP 전원 0 이후에만 `"모험은 끝이 났다."` 출력 + 세션 종료 확인.
