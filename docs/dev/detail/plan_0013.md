# plan_0013 - 유닛 커버리지/통합 러너 검증 및 구조 리스크 정리

## 상태
- 완료 (2026-02-08)

## 목표
- `rule-engine`, `BE-router`를 제외한 서비스 유닛 테스트/커버리지 기준(80%) 충족 여부를 검증한다.
- 통합 테스터(`smoke10`, `endurance20`)가 오류 중단 없이 동작하는지 확인한다.
- 턴 단계별 상태 업데이트와 시퀀스 전이가 의도대로 진행되는지 확인한다.
- 미충족 항목/잠재 리스크/구조적 한계를 문서로 정리한다.

## 작업 단위
1. 사전 자료 확인
- `docs/dev`, 기존 detail plan, 테스트 엔트리 포인트/실행 규약 점검

2. 유닛 테스트 + 커버리지 검증
- 대상: `gm`, `state-manager`, `scenario-service`, `llm-gateway`
- 실행: 각 서비스에서 `pytest + pytest-cov`로 총 커버리지 측정
- 기준: 서비스 총 커버리지 `>= 80%`

3. 통합 러너 검증
- 실행: `PYTHONPATH=tester/src uv run python -m tester.runner smoke10 10`
- 실행: `PYTHONPATH=tester/src uv run python -m tester.runner endurance20 20`
- 확인: 중단 없는 완주, 턴별 상태 출력, 시퀀스 전이/상태 변화(예: HP diff) 반영

4. 결과 문서화
- 성공/실패 항목 구분
- 부족한 부분, 잠재 문제, 현재 구조가 목적 달성에 주는 장애 요인 정리

## 완료 기준 (DoD)
- 유닛 테스트 및 커버리지 결과가 서비스별로 수치와 함께 정리된다.
- 통합 러너 10/20턴 실행 결과가 로그 근거와 함께 정리된다.
- 리스크/구조 이슈 분석 문서가 `docs/dev/detail`에 반영된다.
