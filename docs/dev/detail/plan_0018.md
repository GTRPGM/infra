# plan_0018 - State-manager 인벤토리 표현 불일치 원인 확정

## 상태
- 완료 (2026-02-08)

## 목표
- state-manager의 시나리오 아이템/관계 표현과 세션 인벤토리 표현의 차이를 코드/실행 근거로 확정한다.
- `inventory/update` 성공 이후 `session inventory`/`player.items` 미반영 원인을 재현 가능한 형태로 특정한다.
- 원인별 영향 범위(테스터 로그, BE-router, state-manager 모델)를 정리한다.

## 작업 단위
1. 코드 경로 추적
- scenario inject 시 아이템/관계 저장 경로
- session start 시 플레이어/인벤토리 초기화 경로
- inventory update/read 경로 및 DTO/DB 매핑

2. 재현 실험
- 동일 세션에 update 후 read 즉시 비교
- rule_id/item_id 매핑 및 조인 조건 확인

3. 결론 문서화
- 확정 원인/2차 원인/구조적 리스크 분리
- 수정 우선순위 및 검증 방법 제시

## 완료 기준 (DoD)
- 코드 위치 + API 결과 + 데이터 모델 근거가 함께 제시됨
- “왜 빈 인벤토리로 보이는지”가 단일 문장으로 설명 가능할 정도로 명확해짐

## 실행 결과
1. 확정 원인(1차): `inventory/update`가 no-op
- 라우터는 `request.rule_id`를 `PlayerRepository.update_inventory()`로 전달하지만,
  저장 로직 없이 그대로 반환만 수행함.
  - `services/state-manager/src/state_db/routers/router_UPDATE.py:66`
  - `services/state-manager/src/state_db/repositories/player.py:201`
- 재현:
  - `PUT /state/inventory/update` 성공 응답 직후
  - `GET /state/session/{session_id}/inventory` = `[]`
  - `GET /state/player/{player_id}`의 `items` = `[]`

2. 확정 원인(2차): 실제 동작 API와 BE-router/테스터 경로 불일치
- state-manager에서 실제 인벤토리 반영은 `/state/player/item/earn`(item UUID 기반) 경로로 동작.
- BE-router는 해당 경로를 중계하지 않고 `/state/inventory/update`만 노출.
  - BE-router OpenAPI paths: `['/state/inventory/update']`
- 재현:
  - state-manager 직접 호출: `/state/player/item/earn` 후 inventory/player.items 정상 반영
  - BE-router 호출: `/state/player/item/earn`는 404

3. 확정 원인(3차): 시나리오 표현(관계)과 플레이어 인벤토리 표현(소유)의 분리
- 시나리오 `relations: owns`는 그래프 RELATION이며 플레이어 인벤토리(CONTAINS)와 별개.
- 따라서 NPC/적 owns를 많이 넣어도 플레이어 인벤토리는 자동 증가하지 않음.

4. 구조적 증폭 요인: 엔티티/관계 중복 누적
- 시나리오 주입 시 sequence 루프에서 동일 `scenario_npc_id/scenario_enemy_id`를 시퀀스마다 별도 row로 삽입.
  - `services/state-manager/src/state_db/repositories/scenario.py:107`
- 세션 시작 시 master(session 0) row를 전량 복제.
  - `services/state-manager/src/state_db/Query/BASE/L_npc.sql:10`
  - `services/state-manager/src/state_db/Query/BASE/L_enemy.sql:10`
- 이후 graph relation 초기화가 `tid` 기준 조합 매칭으로 복제되어 중복 edge를 증폭.
  - `services/state-manager/src/state_db/Query/BASE/L_graph.sql:97`

5. 관측 왜곡 요인(테스터)
- 테스터는 inventory 응답을 dict로 가정(`inventory.items`)하여 list 응답을 빈 것으로 출력.
  - `tester/src/tester/runner.py:148`

6. 결론 한 줄
- 현재 빈 인벤토리의 핵심 원인은 “BE-router/테스터가 no-op인 `inventory/update(rule_id)`를 사용하고, 실제 반영 경로인 `player/item/earn(item_uuid)`를 타지 않기 때문”이며, 여기에 시나리오 관계-인벤토리 모델 분리와 관계 중복 구조가 겹쳐 문제를 키운다.
