# plan_0022 - rule-engine 아이템 ID 파싱 정렬 및 전투 종료 정합성 복구

## 상태
- 완료 (2026-02-09)

## 목표
- rule-engine 전투/회복 페이즈에서 문자열 시나리오 아이템 ID(`item-...`)를 처리 가능하도록 정렬한다.
- 전투 HP diff가 정상 반영되어 "적 처치 후 종료" 경로가 자연스럽게 성립하는지 검증한다.

## 작업 단위
1. 파싱 로직 보강
- `combat_node.py`, `recovery_node.py`에서 `int(item.item_id)` 단정 제거.
- 숫자형 RDB ID와 문자열 `scenario_item_id`를 모두 처리하는 조회 경로로 교체.

2. 회귀 방지 테스트
- rule-engine 테스트에 혼합 ID(정수/문자열) 케이스 추가.
- 전투/회복 노드가 예외 없이 diff/로그를 생성하는지 검증.

3. 통합 검증
- rule-engine 단위 테스트 실행.
- BE-router 경유 테스터 재실행으로 시퀀스 진행, 적 HP 감소, 종료 조건을 확인.

## 완료 기준
- `/play/scenario` 500(`invalid literal for int()`)가 재발하지 않는다.
- 테스터에서 전투 단계 이후 적 HP가 실제 감소하고, 종료는 적 생존 시 보류/전멸 시 종료로 동작한다.

## 수행 결과
1. 코드 변경
- `services/rule-engine/src/domains/play/utils/phase_nodes/combat_node.py`
  - `int(item.item_id)` 강제 파싱 제거.
  - 숫자형 아이템 ID는 기존 DB 조회를 유지하고, 문자열 `scenario_item_id` 기반 아이템은 `item_type/meta` fallback으로 전투 보정치를 계산하도록 보강.
- `services/rule-engine/src/domains/play/utils/phase_nodes/recovery_node.py`
  - `int(item.item_id)` 강제 파싱 제거.
  - 문자열 아이템 ID 기반 포션 탐색/회복량 계산 지원.
  - consume relation 매칭을 `match_ids` 기반으로 보강하고, 단일 consume relation 존재 시 fallback 매칭 처리.

2. 테스트 추가/검증

- 실행:
