# plan_0025 - sequence_type 기반 판정 컨텍스트 도입

## 상태
- 완료 (2026-02-09)

## 목표
- 시퀀스 의도(탐색/협상/잠입/전투/결전)를 런타임 판정 컨텍스트로 전달한다.
- GM/RuleEngine가 시퀀스 타입을 공통 인지해 판정 일관성을 높인다.

## 작업 단위
1. state-manager 시나리오 주입/조회 경로에 sequence metadata 보존.
2. GM fetch_state에서 sequence metadata 기반 `sequence_type` 계산.
3. RuleEngine 요청 스키마에 `sequence_type` 전달 필드 추가.
4. RuleEngine 분석 노드가 `sequence_type` 힌트를 우선 반영하도록 보강.
5. six-sequence 평가 payload에 sequence_type 메타 부여.
6. 단위 테스트/회귀 실행 및 결과 문서 반영.

## 수행 결과
- 반영 완료:
  1. `state-manager` 시나리오 주입/조회에 `sequence.metadata` 보존 및 반환 경로 반영.
  2. `gm`에서 현재 시퀀스 메타를 읽어 `sequence_type`을 RuleEngine 요청에 전달.
  3. `rule-engine`이 `sequence_type` 힌트를 우선 사용해 페이즈를 결정하도록 보강.
  4. 6시퀀스 평가 payload에 시퀀스별 `metadata.sequence_type` 명시.
  5. `scenario-service` 주입 변환기에서 `sequence.metadata` 누락 버그 수정
     (`debug/inject-save` 경로 포함).
