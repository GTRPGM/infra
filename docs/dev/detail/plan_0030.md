# plan_0030 - 프로젝트 전체 구조도(서비스 및 db 단위) 작성 계획

## User Requirements
- 프로젝트의 전체 서비스 및 DB 구조를 시각화해야 함.
- 주석 없는 Mermaid 다이어그램을 사용해야 함.
- 전체 구조도와 각 서비스별 세부 아키텍처가 포함되어야 함.

## Goal
- 서비스 간의 의존성, 데이터 흐름, DB 연결 구조를 한눈에 파악할 수 있는 아키텍처 문서 작성.

## Scope
- In:
  - 전체 시스템 서비스 맵 (Mermaid)
  - 서비스별(BE-router, gm, state-manager, scenario-service, rule-engine, llm-gateway) 내부 구조도
  - 서비스-DB 간의 논리적 연결도
- Out:
  - 인프라 하드웨어 수준의 상세 구성도 (VPC, Subnet 등)

## Implementation Plan
- Step 1: `docs/dev/architect/architecture_v0.1.0.md` 파일을 생성하고 전체 시스템 레벨의 Mermaid 다이어그램 작성.
- Step 2: 각 개별 서비스의 폴더를 스캔하여 내부 컴포넌트 구조를 파악하고 세부 아키텍처 다이어그램 추가.
- Step 3: DB 논리 분리 현황(Postgres 내 각 서비스용 DB)을 반영한 스토리지 구조도 작성.

  - 전체 문서를 읽었을 때 실제 `docker-compose.local.yml`의 서비스 구성과 일치하는지 대조.
- Expected:
  - 모든 서비스와 DB 연결이 실제 코드 및 설정과 일치함.
