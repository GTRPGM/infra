# plan_0020 - seq-4 전이 고착 수정

## 상태
- 완료 (2026-02-08)

## 목표
- 고도화 시나리오 실행 시 `seq-4`에서 전이가 멈추는 원인을 코드/로그로 확정한다.
- 원인 지점의 전이 로직을 수정해 `seq-5`, `seq-6`까지 진행 가능하게 만든다.
- 유닛 테스트와 테스터 재실행으로 수정 효과를 검증한다.

## 작업 단위
1. 원인 분석
- GM/Scenario-Service/State-manager 로그에서 전이 요청/응답 추적
- 전이 조건 판정 코드와 시퀀스 매핑 코드 확인

2. 코드 수정
- 전이 조건/매핑 실패 지점 보정
- 필요 시 관측 로그(전이 실패 이유) 강화

3. 검증
- 관련 유닛 테스트 실행
- `six_sequence_three_act_eval` 테스터 재실행

## 완료 기준 (DoD)
- `seq-4` 고착이 재현되지 않고 이후 시퀀스로 진행됨
- 테스트 통과 및 결과 문서화 완료

## 수행 결과
1. 원인 확정
- `scenario-service /api/v1/check/validate`에서 말단 시퀀스(`seq-2`, `seq-4`)에 대해 LLM이 동일/역방향 `next_seq_id`를 반환.
- `_guard_non_forward_sequence_transition`가 이를 차단하며 `next_seq_id=None`으로 변경.
- 이어서 `terminal sequence reached: should_end`가 설정되어 액트 경계 전이가 소실됨.

2. 코드 수정
- 파일: `services/scenario-service/src/scenario/core/engine/scenario_engine.py`
- 변경 사항:
  - exit trigger 매칭 시, 동일 액트 내 다음 시퀀스가 없어도 `is_triggered=true`를 유지.
  - 액트 경계(`act-n` 말단 시퀀스)에서 `next_act_id/next_seq_id` 자동 보정 추가.
  - guard 이후 전이값이 사라진 경우, exit trigger가 실제 매칭되면 `act-(n+1)` 첫 시퀀스로 복구하는 후처리 추가.

3. 테스트
- 유닛 테스트:
  - `cd services/scenario-service && uv run --group dev python -m pytest tests/test_plan_0011_transition_fallback.py -q`
  - 결과: `8 passed`
  - `cd services/scenario-service && uv run --group dev python -m pytest tests/test_api.py -q`
  - 결과: `9 passed`
- 런타임 API 검증:
  - `act-1/seq-2` 입력 시 `next_act_id=act-2`, `next_seq_id=seq-3` 확인
  - `act-2/seq-4` 입력 시 `next_act_id=act-3`, `next_seq_id=seq-5` 확인
- 테스터:
  - `PYTHONPATH=tester/src uv run python -m tester.runner sixacteval6_fix5 6 six_sequence_three_act_eval`
  - 결과 로그: `log/test_output_sixacteval6_fix5_20260208_205333.log`
  - 전이 확인: `seq-1 -> seq-2 -> seq-3 -> seq-4 -> seq-5 -> seq-6`
  - 실행 중단/예외 없음, 테스트 정상 종료.

## 잔존 이슈
- 아이템/인벤토리 표시는 여전히 `unknown` 중심이며, 시나리오 엔티티 아이템과 state-manager 인벤토리 표현 매핑 불일치가 남아 있음.
- 최종 시퀀스(`seq-6`)에서 `should_end`가 실제 세션 종료로 이어지지 않고 반복 진행되는 케이스가 존재함
  (적 HP/생존 판정과 종료 게이트 결합 이슈).
