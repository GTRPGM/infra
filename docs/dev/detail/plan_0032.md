# plan_0032 - NPC 및 적 엔티티의 관계 기반 아이템 소유 및 드롭 시스템 구현

## User Requirements
- NPC와 적(Enemy)의 아이템 소유를 단순 리스트(`dropped_items`)가 아닌 그래프 관계 기반으로 관리해야 함.
- 시나리오 주입 시점에 엔티티별 아이템 소유 관계가 생성되어야 함.
- 엔티티가 비활성화(처치/퇴장)되면 소유하던 아이템이 해당 장소(필드)에 남도록 자동 업데이트되어야 함.

## Goal
- 플레이어와 동일한 인벤토리/소유 모델을 NPC 및 적에게도 적용하여 데이터 일관성 확보.
- 아이템 수량(quantity) 개념을 엔티티 소유 관계에 도입.
- 처치 시 아이템 드롭 로직을 그래프 관계 전이 방식으로 자동화.

## Scope
- In: `state-manager` 그래프 스키마, `ScenarioRepository` 주입 로직, `EntityRepository` 비활성화 로직, `scenario-service` 모델 및 주입 페이로드.
- Out: 플레이어 인벤토리 UI 변경, 아이템 사용 효과(Effect) 구현.

## Implementation Plan
- Step 1: `state-manager`의 `ScenarioRepository.inject_scenario`에서 NPC/Enemy와 Item 간의 `[:OWNS]` 또는 `[:CONTAINS]` 관계를 생성하도록 수정.
- Step 2: `scenario-service`의 시나리오 생성 스키마에 엔티티별 아이템 소유 정보를 포함하도록 모델링 변경.
- Step 3: `state-manager`에서 엔티티 처치(`defeat_enemy`) 또는 퇴장(`depart_npc`) 시, 해당 엔티티가 가진 아이템 관계를 현재 시퀀스 노드(`:Sequence`)와의 관계(예: `[:HAS_ITEM]`)로 전환하는 트리거 또는 로직 구현.
- Step 4: `state-manager`의 `Item` 조회 시 필드 아이템과 엔티티 소유 아이템을 명확히 구분하여 반환하도록 쿼리 정렬.

## Unit Test Plan (must)
- What to test: NPC/Enemy 소유 관계 생성 여부, 엔티티 비활성화 시 아이템 관계 전이 여부.
- Test cases:
  - 시나리오 주입 후 NPC가 지정된 아이템을 `quantity`와 함께 소유하고 있는지 확인.
  - `defeat_enemy` 호출 후 Enemy가 소유하던 아이템이 시퀀스(필드)로 이동했는지 확인.
- Command:
  - uv run pytest services/state-manager/test

## E2E Plan (must)
- Scenario: 시나리오 생성 -> 주입 -> 적 처치 -> 처치된 적이 있던 장소에서 아이템 획득 가능 여부 확인.
- Expected: 처치 전에는 적이 소유하고, 처치 후에는 필드 아이템으로 상태가 변경됨.
- How to run: `tester/src/tester/runner.py` 기반 커스텀 테스트 시나리오 실행.
- Validation: `get_state_snapshot` API를 통해 아이템의 관계 주체 변화 확인.
