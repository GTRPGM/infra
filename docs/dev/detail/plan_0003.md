# plan_0003 - GM 계약 정렬 (중계기 역할 강화)

## 목표
- GM을 "판단+중계 오케스트레이터"로 정렬한다.
- 외부 서비스에 보내는/받는 데이터가 각 서비스 계약과 정확히 일치하도록 보정한다.

## 작업 단위
1. Outbound 계약 정렬
- 대상: `services/gm/src/gm/plugins/external/http_client.py`
- 조치:
  - Rule 요청 payload 필드/타입을 Rule 계약과 정확히 일치
  - Scenario validate 요청(`scenario_id`, `act_id`, `seq_id`, `user_input`) 정규화
  - State commit/update 요청 payload 정규화

2. Inbound 파싱 정렬
- 대상: `services/gm/src/gm/plugins/external/http_client.py`, `services/gm/src/gm/schemas/*`
- 조치:
  - wrapped/flat 응답 케이스 표준 처리
  - 누락 필드에 대한 안전한 fallback + 명확한 오류 분리

3. 코어 파이프라인 안정화
- 대상: `services/gm/src/gm/core/engine/game_engine.py`
- 조치:
  - 외부 계약 mismatch 시 조기 실패(fail-fast) 지점 정의
  - 전이(`next_act_id`, `next_seq_id`) 적용 조건 강화
  - 내러티브 생성은 commit 성공 후 실행 불변 조건 유지

4. 관찰성 강화
- 대상: `services/gm/src/gm/core/engine/game_engine.py`
- 조치:
  - 노드별 요청/응답 요약 로그 표준화(민감정보 제외)
  - 계약 오류 코드 분류(Validation/Upstream/Timeout)

## 완료 기준 (DoD)
- GM이 Rule/Scenario/State와 통신 시 필드 mismatch로 인한 런타임 오류가 없어야 함.
- GM 내부 로직은 유지하면서도 외부 계약 적합성 테스트가 통과해야 함.

## 실행 결과 (2026-02-07)
- Outbound 계약 정렬 완료:
  - State 전이 요청을 State 스키마에 맞게 변경
    - `update_act`: `new_act`, `new_act_id`, `new_sequence_id`
    - `update_sequence`: `new_sequence`, `new_sequence_id`
- Inbound 파싱 정렬 완료:
  - Rule/Scenario/State 응답에서 wrapped(`{status,data}`)/flat 동시 처리.
- 코어 파이프라인 안정화:
  - `next_act_id`만 있고 `next_seq_id`가 없으면 fail-fast로 즉시 오류 처리.
  - 내러티브는 기존과 동일하게 commit 이후에만 생성.
- 관찰성 보강:
  - Rule/Scenario 클라이언트 오류 로그에 `session/scenario/act/seq` 핵심 컨텍스트 포함.
