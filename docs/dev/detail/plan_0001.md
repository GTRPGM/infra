# plan_0001 - 기준 계약 동결 및 정합성 감사 프레임

## 목표
- `state-manager`/`rule-engine`를 기준 계약(immutable contract)으로 고정하고, 나머지 서비스의 정합성 점검 기준을 문서화한다.

## 고정 경계
- `rule-engine`: **수정 금지**.
- `state-manager`: **오류가 확인된 경우에만 최소 수정**, 기본은 비수정.
- 변경 허용: `scenario-service`, `gm`, `BE-router`, `tester/src/tester`, 문서/테스트.

## 작업 단위
1. 계약 원천(Source of Truth) 고정
- State 시나리오 주입 계약: `services/state-manager/src/state_db/schemas/scenario.py`
- State 주입 저장 동작: `services/state-manager/src/state_db/repositories/scenario.py`
- Rule 입력/출력 계약: `services/rule-engine/src/domains/play/dtos/play_dtos.py`, 관련 router/service

2. 인터서비스 계약 인벤토리 작성
- Scenario -> State inject payload 필드 매핑표 작성
- GM -> Rule/Scenario/State 송수신 필드 매핑표 작성
- BE-router -> GM 프록시 계약 점검표 작성

3. 정합성 등급 기준 정의
- `Blocker`: 422/500 유발, ID 참조 붕괴, 필수 필드 누락
- `Major`: 타입/의미 불일치(동작은 하나 품질 저하)
- `Minor`: 불필요 필드, 네이밍 비일관성

## 산출물
- 계약 인벤토리 문서(경로: `docs/dev/detail/ref_contract_matrix.md` 예정)
- 서비스별 결함 리스트(우선순위/소유 서비스 포함)
