# plan_0031 - 플레이어-NPC 관계 생성 누락 원인 파악 및 정합성 보강

## User Requirements
- 플레이어-NPC 관계가 제대로 형성되지 않는 문제의 근본 원인을 파악해야 함.
- 각 서비스(Rule Engine, GM, State Manager, Scenario Service) 내에 유닛 테스트를 추가하여 데이터 흐름 상의 누락 지점을 확인해야 함.
- 룰엔진-GM, GM-스테이트 매니저 사이의 통신 스키마에 관계 정보가 포함되어 있는지 확인해야 함.

## Goal
- 서비스 간 관계(Relation) 데이터 전달 체계의 정합성을 확보하고, 신규 관계 생성 기능이 정상 동작함을 보증한다.

## Scope
- In:
  - `rule-engine`: 탐험/대화 판정 시 `relation` 생성 및 반환 로직.
  - `gm`: 룰엔진 결과 수신 후 `state-manager` 커밋 요청으로의 매핑.
  - `state-manager`: 관계 추가/조회 API 및 DB 처리.
  - `scenario-service`: 시나리오 로직 내 관계 삭제 여부 확인.
- Out:
  - 플레이어 외 엔티티 간의 관계 설정 (이번 범위에서는 플레이어-NPC 관계에 집중).

## Implementation Plan
- Step 1: **Schema Verification**: Rule Engine -> GM, GM -> State Manager 간 DTO에 `relations` 필드가 누락되었는지 전수 조사.
- Step 2: **Rule Engine Investigation**: 탐험/대화 노드에서 관계 생성 조건(우호도 임계치 등) 및 리턴 포맷 검증 유닛 테스트 작성.
- Step 3: **GM Service Investigation**: `Turn Pipeline`에서 룰엔진 응답을 `State Manager`의 `commit` 요청으로 변환할 때 관계 정보가 보존되는지 테스트.
- Step 4: **State Manager Investigation**: 관계 생성/조회 유닛 테스트를 통해 DB 및 Graph 연동 확인.
- Step 5: **Scenario Service Audit**: 시나리오 판정 프롬프트 또는 로직에서 관계를 무효화하거나 삭제하는 코드가 있는지 검색 및 확인.

## Unit Test Plan (must)
- What to test:
  - Rule Engine: 판정 결과 DTO에 신규 관계 포함 여부.
  - GM: 룰엔진 응답 -> State Manager 요청 변환 정합성.
  - State Manager: 관계 CRUD 기능.
- Test cases:
  - 관계가 없는 NPC와 상호작용 후 관계 생성 응답이 발생하는지.
  - 생성된 관계가 State Manager에 정상 반영되는지.
- Command:
  - `uv run pytest services/rule-engine/tests`
  - `uv run pytest services/gm/tests`
  - `uv run pytest services/state-manager/tests`

## E2E Plan (must)
- Scenario: 플레이어가 처음 만나는 NPC와 대화하여 관계를 형성하고, 이후 상태 조회 시 해당 관계가 존재하는지 확인.
- Expected: 상태 요약 또는 상세 조회 시 플레이어-NPC 관계가 `friendly` 등으로 표시됨.
- How to run: `PYTHONPATH=tester/src uv run python -m tester.runner relation_test 5`
- Validation: 턴 로그 및 `/state/session/{id}/relations` 응답 확인.
